# 层级定义文档

> 定义 FoxChat 记忆架构的六层职责边界、注入方式与实现状态
> 版本：第一版
> 更新时间：2026-04-27

---

## 一、六层架构总览

```text
A. 常驻稳定层
   A1. 静态锚点
      - soul
      - core_anchor
      - character_card

   A2. 稳定画像 / 关键边界
      - user_profile
      - 长期偏好
      - 永不衰减禁忌
      - 高优先级边界

B. 当前工作状态层
   - emotion
   - relation_state
   - current_focus
   - unfinished_items
   - interaction_mode

T. 时间节点层（运行时激活，不常驻注入）
   - future commitments
   - due follow-ups
   - time-bound reminders

C. 历史事件检索层
   - 过去重要事件
   - 承诺 / 冲突 / 共同经历
   - 旧话题锚点

D. 最近对话窗口
   - 最近 4~6 轮原始消息
```

## 二、各层职责边界

### A1. 静态锚点

| 属性 | 说明 |
|------|------|
| **职责** | 固定角色身份、风格与底线 |
| **内容** | soul、core_anchor、character_card |
| **更新频率** | 基本不变 |
| **注入方式** | 每轮常驻 |
| **权重** | 最高 |

### A2. 稳定画像 / 关键边界

| 属性 | 说明 |
|------|------|
| **职责** | 表达长期稳定的用户画像、偏好与禁忌 |
| **内容** | user_profile、长期偏好、永不衰减禁忌、高优先级边界 |
| **更新频率** | 随总结缓慢更新 |
| **注入方式** | 每轮常驻，不应依赖检索召回 |
| **提升条件** | 连续2个summary batch提取同一结论，或30轮内>=3次一致表达 |

### B. 当前工作状态层

| 属性 | 说明 |
|------|------|
| **职责** | 表达"现在是什么局面，这一轮应如何继续回应" |
| **内容** | emotion、relation_state、current_focus、unfinished_items、interaction_mode |
| **更新频率** | 高于A2，按轮次过期衰减 |
| **注入方式** | 每轮常驻，注入摘要而非原始JSON |
| **过期机制** | 各字段独立 expire_rounds |

### T. 时间节点层

| 属性 | 说明 |
|------|------|
| **职责** | 保存"未来某个时间点应该重新浮现"的事项，到期时激活 |
| **内容** | future commitments、due follow-ups、time-bound reminders |
| **更新频率** | 按到期条件触发 |
| **注入方式** | **不常驻注入**，按当前时间与节点条件在运行时激活 |
| **触发逻辑** | 到期后进入B层或辅助触发C层检索 |

**重要说明：** T层不是A2/B/C/D的替代层，而是一个横向运行时机制：
- 不负责长期画像，不进入A2
- 不直接等同于当前状态，到期时激活到B层或辅助触发C层
- 不等同于历史事件检索，触发依据是"时间到没到"而非"语义像不像"

```text
过去提到的未来事项
   ↓
生成 time node
   ↓
pending 挂起
   ↓
到期后 runtime 激活
   ↓
进入 current_state / 辅助历史检索触发
```

### C. 历史事件检索层

| 属性 | 说明 |
|------|------|
| **职责** | 按需回忆与当前输入相关的过去事件 |
| **内容** | 过去重要事件、承诺/冲突/共同经历、旧话题锚点 |
| **更新频率** | 按总结写入 |
| **注入方式** | 检索式、限量注入，不应承担当前状态职责 |
| **触发条件** | 回忆信号、current_focus相关、time node到期辅助触发 |

### D. 最近对话窗口

| 属性 | 说明 |
|------|------|
| **职责** | 保证近程上下文、语气、指代与连贯性 |
| **内容** | 最近4~6轮原始消息 |
| **更新频率** | 每轮滚动 |
| **注入方式** | 作为History Messages |
| **特点** | 工作窗口，不属于长期记忆 |

## 三、注入顺序

**最终注入顺序：**

```text
【头部·最高权重底线】     A1 静态锚点
【头部后段·高权重长期约束】 A2 稳定画像 / 关键边界
【中部·补充信息】          C 历史事件检索结果
【近尾部·当前局面核心】    B 当前工作状态层
【紧邻输入·上下文连续性】  D 最近对话窗口
【最尾部·最高即时权重】    用户当前输入
```

**工程实现建议表达：**

```text
System Block 1：静态锚点 + 硬边界
System Block 2：稳定画像 / 长期禁忌
System Block 3：历史事件检索结果
System Block 4：当前工作状态 + 冲突优先级规则
History Messages：最近对话窗口
Human Message：用户当前输入
```

## 四、Token 预算原则

> **标注：以下预算为观测指标，不作为第一版设计前置硬约束**
> 第一版先看 correctness，再看 token

| 层级 | 预算范围 | 说明 |
|------|----------|------|
| A1 静态锚点 | 固定预算 | 内容基本不变 |
| A2 稳定画像/边界 | 固定预算 | 常驻但不膨胀 |
| B 当前状态层 | 80~150 tokens | 状态摘要本体；含冲突规则时上限250 |
| C 历史事件检索 | 300~600 tokens | 2~4条事件 |
| D 最近对话窗口 | 1200~1600 tokens | 最近4~6轮 |

**注入原则：**
- 存储用JSON，注入给模型时转成短摘要
- 明确冲突规则：硬边界 > 当前状态 > 历史事件 > 近程模糊暗示
- 不默认把全部元数据直接注入Prompt
